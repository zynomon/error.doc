1. Disk Partitions and Filesystems

Understanding Partitions

A partition is a logical division of a physical hard disk. Linux organizes storage into partitions, each of which can hold a filesystem. The standard partitioning scheme for Linux includes:

  • / (root): The main partition where the operating system lives
  • /home: Separate partition for user data (makes reinstalling easier)
  • /boot: Contains bootloader and kernel files
  • swap: Used as virtual memory when RAM fills up
  • /var: For logs and variable data (good to keep separate)
  • /tmp: Temporary files (often mounted as tmpfs in RAM)

Common Filesystems

  • ext4: The default for most Linux distributions. Mature, reliable, and well-supported
  • XFS: Excellent for large files and high performance, common on servers
  • btrfs: Modern filesystem with snapshots, compression, and checksumming
  • ZFS: Advanced filesystem with volume management, not included in mainline kernel
  • FAT32/NTFS: Windows compatibility
  • tmpfs: In-memory filesystem (files disappear on reboot)

Partition Management Commands

View disk information:

lsblk # list all block devices sudo fdisk -l # detailed partition info df -h # disk usage of mounted filesystems sudo parted -l # partition table information

Create and manage partitions:

sudo fdisk /dev/sda # interactive partition editor sudo gdisk /dev/sda # GPT partition editor sudo parted /dev/sda # advanced partition tool sudo mkfs.ext4 /dev/sda1 # format partition as ext4 sudo mkfs.xfs /dev/sda2 # format as XFS sudo mkswap /dev/sda3 # create swap space sudo swapon /dev/sda3 # enable swap

Mounting Filesystems

Before using a partition, you need to mount it to a directory:

sudo mount /dev/sda1 /mnt # mount partition to /mnt sudo umount /mnt # unmount mount # view all mounted filesystems sudo mount -o remount,rw / # remount as read-write

Persistent mounts: Add entries to /etc/fstab to mount automatically at boot:

# device    mountpoint  fstype  options      dump pass
UUID=1234   /           ext4    defaults     0    1
UUID=5678   /home       ext4    defaults     0    2
UUID=abcd   none        swap    sw           0    0

2. APT Package Management

APT (Advanced Package Tool) is the package manager for Debian and its derivatives like Ubuntu. It handles installation, removal, and updates of software.

Basic APT Commands

sudo apt update # update package list from repositories sudo apt upgrade # upgrade all installed packages sudo apt full-upgrade # upgrade with dependency resolution sudo apt install package # install a package sudo apt remove package # remove package (keep config files) sudo apt purge package # remove package and config files sudo apt autoremove # remove unused dependencies apt search keyword # search for packages apt show package # show package details apt list --installed # list installed packages apt list --upgradable # list packages that can be upgraded

Advanced Package Operations

sudo apt download package # download .deb file without installing sudo apt source package # download source code sudo apt build-dep package # install build dependencies sudo apt install ./file.deb # install local .deb file sudo apt-mark hold package # prevent package from being upgraded sudo apt-mark unhold package # remove hold apt-cache depends package # show package dependencies apt-cache rdepends package # show reverse dependencies

Managing Repositories

Software sources are defined in /etc/apt/sources.list and files in /etc/apt/sources.list.d/. A typical line looks like:

deb http://deb.debian.org/debian bookworm main contrib non-free

Components:

  • main: Free software officially supported
  • contrib: Free software that depends on non-free components
  • non-free: Proprietary software
  • non-free-firmware: Proprietary firmware
sudo add-apt-repository ppa:user/name # add Personal Package Archive sudo add-apt-repository --remove ppa:user/name # remove PPA sudo apt update # always update after adding repos

3. sudo and Privilege Escalation

sudo (superuser do) allows authorized users to run commands with elevated privileges. It is safer than logging in as root because it logs all commands and uses your own password.

Using sudo

sudo command # run command as root sudo -i # interactive root shell sudo -s # root shell preserving environment sudo -u username command # run command as another user sudo -l # list allowed commands for your user sudo -k # invalidate cached credentials sudo -v # update timestamp (extend timeout)

The first time you use sudo in a session, it prompts for your password. The privilege lasts for 15 minutes by default.

The sudoers File

Never edit /etc/sudoers directly. Use visudo, which checks syntax before saving:

sudo visudo

Example sudoers entries:

# User specification
john    ALL=(ALL:ALL) ALL              # john can run any command
alice   ALL=(ALL) /usr/bin/systemctl   # alice can only run systemctl

# Group specification (using % prefix)
%sudo   ALL=(ALL:ALL) ALL              # members of sudo group can run any command
%developers ALL=(ALL) NOPASSWD: ALL    # developers group, no password required

# Command aliases
Cmnd_Alias NETWORKING = /sbin/ifconfig, /bin/ping, /usr/bin/nmap
bob     ALL=(ALL) NETWORKING           # bob can only run networking commands

sudo Best Practices

  • Use sudo for individual commands, not entire shells when possible
  • Never run graphical applications with sudo (use pkexec or gksudo instead)
  • The sudo group (Debian/Ubuntu) or wheel group (RHEL/Fedora) controls who gets sudo access
  • Check logs in /var/log/auth.log to see who used sudo and what they did

4. Users and Groups

Linux is a multi-user system. Every process runs as a specific user and belongs to one or more groups.

User Management Commands

# Create users sudo useradd john # basic user creation sudo useradd -m john # create with home directory sudo useradd -m -s /bin/bash john # specify shell sudo adduser john # interactive (Debian/Ubuntu) # Modify users sudo usermod -aG sudo john # add to sudo group (append) sudo usermod -L john # lock account sudo usermod -U john # unlock account sudo usermod -s /bin/zsh john # change shell # Delete users sudo userdel john # keep home directory sudo userdel -r john # delete home directory too # Password management passwd # change your own password sudo passwd john # change another user's password sudo passwd -l john # lock user sudo passwd -e john # force password change on next login

Group Management Commands

# Create and delete groups sudo groupadd developers # create group sudo groupdel developers # delete group # Modify groups sudo groupmod -n newname oldname # rename group sudo usermod -aG developers john # add user to group sudo gpasswd -d john developers # remove user from group # View group information groups # show current user's groups groups john # show john's groups id john # show user and group IDs getent group developers # show group members

File Permissions

Every file has three permission sets: Owner (u), Group (g), Others (o). Each set has three permissions: Read (r=4), Write (w=2), Execute (x=1).

Numeric (octal) method:

chmod 755 script.sh # rwxr-xr-x (owner: all, group: rx, others: rx) chmod 644 file.txt # rw-r--r-- (owner: rw, group: r, others: r) chmod 700 private.sh # rwx------ (owner only) chmod 600 ~/.ssh/id_rsa # rw------- (SSH private key)

Symbolic method:

chmod u+x script.sh # add execute for owner chmod g+w file.txt # add write for group chmod o-r private.txt # remove read for others chmod a+x program # add execute for all chmod u=rwx,g=rx,o=r file # set exact permissions chmod -R 755 directory/ # recursive permission change

Special Permissions

# setuid (runs as file owner) chmod u+s /usr/bin/passwd # 4755 chmod 4755 program # numeric setuid # setgid (new files inherit group) chmod g+s /shared/directory # 2755 chmod 2755 directory # numeric setgid # sticky bit (only owner can delete) chmod +t /tmp # 1777 chmod 1777 /tmp # numeric sticky

View permissions:

ls -l file.txt # -rw-r--r-- 1 alice developers 1024 Jan 23 10:00 file.txt

5. Bash Shell

Bash (GNU Bourne Again Shell) is the default shell on most Linux systems. It processes commands, runs programs, and provides scripting capabilities.

Bash Operators

Pipe (|): Takes output of the left command and uses it as input to the right command

ls -la | grep ".txt" # list files and filter for .txt ps aux | grep firefox # find firefox processes cat file.txt | wc -l # count lines in file

Logical AND (&&): Run second command only if first succeeds

mkdir newdir && cd newdir # create and enter only if creation worked sudo apt update && sudo apt upgrade # update then upgrade

Logical OR (||): Run second command only if first fails

apt search alien || apt search human # try one search, fallback to another command_not_found || echo "command failed"

Background (&): Run command in background

firefox & # launch firefox and get prompt back long_running_task & # run task in background

Command substitution ($() or ``): Use output of command as argument

echo "Today is $(date)" # embed date in string cd $(dirname $(which python)) # go to python's directory

Input and Output Redirection

command > file # redirect stdout to file (overwrite) command >> file # append stdout to file command 2> error.log # redirect stderr to file command &> output.log # redirect both stdout and stderr command < input.txt # use file as input command 2>&1 # redirect stderr to stdout

Variables and Environment

# Local variables myvar="hello" # no spaces around = echo $myvar # use $ to access readonly myvar # make variable read-only # Environment variables export PATH=$PATH:/my/bin # add to PATH echo $PATH # view PATH env # list all environment variables set # list all variables # Special variables echo $? # exit status of last command echo $$ # current shell PID echo $0 # script name echo $1 # first argument echo $@ # all arguments

Job Control

jobs # list background jobs fg %1 # bring job 1 to foreground bg %2 # resume job 2 in background Ctrl+Z # suspend current foreground job Ctrl+C # terminate current foreground job

6. GNU Coreutils Command Reference

These are the everyday commands you will use constantly. They come from the GNU coreutils package.

rm (remove)

rm file.txt # delete file rm -r directory/ # recursive delete rm -f file.txt # force delete (no prompt) rm -i file.txt # interactive (ask before delete) rm -rf /tmp/* # dangerous - delete everything in /tmp rm -v file.txt # verbose (show what's deleted)

Warning: Combining -f with -i is contradictory. -f overrides -i, so prompts will not appear.

ls (list)

ls # list files ls -l # long format ls -a # show hidden files (starting with .) ls -la # long format with hidden ls -lh # human-readable sizes ls -lt # sort by modification time ls -ltr # reverse time order ls -R # recursive (show subdirectories) ls -l /etc # list /etc directory

cp (copy)

cp file1 file2 # copy file1 to file2 cp -r source/ dest/ # recursive copy (directories) cp -i file1 file2 # interactive (prompt before overwrite) cp -v file1 file2 # verbose cp -p file1 file2 # preserve timestamps and permissions cp -u file1 file2 # update (copy only if source is newer) cp file1 file2 file3 dest/ # copy multiple files to directory

mv (move)

mv oldname newname # rename file mv file.txt /backup/ # move to directory mv -i file.txt /backup/ # interactive (prompt before overwrite) mv -v file.txt /backup/ # verbose mv -n file.txt /backup/ # no-clobber (don't overwrite) mv *.txt documents/ # move all txt files

cat (concatenate)

cat file.txt # display file cat file1 file2 > combined # merge files cat -n script.sh # show with line numbers cat -s file.txt # squeeze repeated empty lines cat << EOF > newfile.txt # here document content here EOF

mkdir (make directory)

mkdir newdir # create directory mkdir -p one/two/three # create nested directories mkdir -v newdir # verbose mkdir -m 755 public # create with specific permissions

touch

touch newfile.txt # create empty file touch existing.txt # update timestamp touch -c file.txt # don't create if doesn't exist touch -t 202401011200 old.txt # set specific timestamp touch file{1..10}.txt # create multiple files

chmod (change mode)

chmod 755 script.sh # rwxr-xr-x chmod 644 file.txt # rw-r--r-- chmod 600 ~/.ssh/id_rsa # rw------- chmod u+x script.sh # add execute for owner chmod g+w file.txt # add write for group chmod -R 755 directory/ # recursive chmod 4755 program # setuid (runs as owner) chmod 1777 /tmp # sticky bit

chown (change owner)

sudo chown alice file.txt # change owner sudo chown alice:staff file.txt # change owner and group sudo chown :staff file.txt # change group only sudo chown -R alice:staff /home/alice # recursive

grep (global regular expression print)

grep "text" file.txt # search for text grep -i "text" file.txt # case insensitive grep -r "text" directory/ # recursive search grep -v "exclude" file.txt # exclude matching lines grep -n "text" file.txt # show line numbers grep -l "text" *.txt # list filenames with matches grep -c "text" file.txt # count matches ps aux | grep firefox # search processes

find (search for files)

find / -name "file.txt" # find by name find . -type f # find all files find . -type d # find all directories find . -size +100M # files larger than 100MB find . -mtime -7 # modified in last 7 days find . -user john # owned by john find . -exec chmod 755 {} \; # execute command on results find . -name "*.tmp" -delete # find and delete

7. Regular Expressions in Linux

Regular expressions (regex) are powerful pattern matching tools used in grep, sed, awk, and many other commands.

Basic regex metacharacters:

  • . - any single character
  • * - zero or more of previous character
  • + - one or more of previous character
  • ? - zero or one of previous character
  • ^ - start of line
  • $ - end of line
  • [abc] - any character in set
  • [^abc] - any character not in set
  • [a-z] - range of characters
  • \| - alternation (or)
  • \(\) - grouping
  • \{n\} - exactly n times
  • \{n,\} - at least n times
  • \{n,m\} - between n and m times

Common regex examples:

grep "^root" /etc/passwd # lines starting with root grep "bash$" /etc/passwd # lines ending with bash grep "[0-9]\{3\}-[0-9]\{3\}" file.txt # phone number pattern grep "error\|fail" log.txt # lines with error or fail grep -E "https?://" file.txt # http or https URLs (extended regex) sed 's/old/new/g' file.txt # replace all occurrences

Extended regex (grep -E or egrep): Removes the need for backslashes

grep -E "(abc|def)" file.txt # abc or def grep -E "a+" file.txt # one or more a's grep -E "a?" file.txt # zero or one a

Linux is built on these fundamentals. Partitions structure your storage, apt manages your software, sudo handles security, users and groups control access, and bash ties everything together with its operators and commands. Master these, and you will be comfortable on any Linux system.
