Driver Management on Linux

Drivers are what let your hardware talk to your operating system. On Linux, most drivers are built right into the kernel, which means they work out of the box. But sometimes you need to install, update, or troubleshoot drivers for specific hardware. This guide walks you through how drivers work and how to manage them.

1. How Drivers Work on Linux

Unlike Windows where you usually download drivers from manufacturer websites, Linux drivers work differently. Most drivers are included in the Linux kernel itself. When you update your kernel, you also update your drivers. This is why keeping your system updated is so important.

Drivers on Linux come in three forms:

  • Built into the kernel: These load automatically when you boot. Most hardware falls into this category.
  • Loadable kernel modules: These are driver files stored in /lib/modules that load when the hardware is detected.
  • User-space drivers: These run outside the kernel. Graphics drivers (like Mesa) and printer drivers (CUPS) are examples.

Some hardware needs proprietary (non-free) drivers or firmware that cannot be included in the main kernel for licensing reasons. Those need to be installed separately.

2. Checking What Hardware You Have

Before managing drivers, you need to know what hardware is in your system. Here are the tools to find out.

List all PCI devices (graphics cards, network adapters, storage controllers):

lspci

For more detail, add -v or -k to see which kernel driver is in use:

lspci -k

If you only want to see your graphics card:

lspci | grep VGA

List USB devices:

lsusb

Show detailed system hardware information:

sudo apt install hwinfo -y sudo hwinfo --short

Show CPU information:

cat /proc/cpuinfo

Show what kernel modules are currently loaded:

lsmod

3. Graphics Drivers

Graphics drivers are the most common drivers people need to manage. They have the biggest impact on gaming, video playback, and desktop performance.

NVIDIA

NVIDIA cards need proprietary drivers. The open-source Nouveau driver works for basic desktop use but is not suitable for gaming or 3D work.

Check what NVIDIA card you have:

lspci | grep -i nvidia

Install NVIDIA drivers:

sudo apt update sudo apt install nvidia-driver firmware-misc-nonfree

For newer cards or better performance, you may want a newer driver version. On systems that support PPAs:

sudo add-apt-repository ppa:graphics-drivers/ppa sudo apt update sudo apt install nvidia-driver-550

After installation, reboot:

sudo reboot

Check if NVIDIA drivers are working:

nvidia-smi

This should show your GPU and driver version.

Switch between NVIDIA and integrated graphics (on laptops): NVIDIA Optimus laptops need special handling. Install the switcher:

sudo apt install nvidia-prime

Then use prime-select to choose:

sudo prime-select nvidia # or sudo prime-select intel

Log out and back in for changes to take effect.

AMD / Radeon

AMD cards use the open-source drivers that come with the kernel. They are called amdgpu (for newer cards) or radeon (for older cards). These work very well, even for gaming.

Check which driver is in use:

lspci -k | grep -A 2 -i vga

Look for "Kernel driver in use: amdgpu" or "radeon".

Install Vulkan support (needed for gaming and Proton):

sudo apt install mesa-vulkan-drivers mesa-vulkan-drivers:i386

For newer AMD cards, you may want a newer kernel: On older distributions, consider using a newer kernel or backports to get the latest amdgpu driver improvements.

sudo apt install linux-image-amd64-backports

Check Mesa version (the OpenGL/Vulkan userspace driver):

glxinfo | grep "OpenGL version"

Intel Integrated Graphics

Intel graphics are fully open-source and included. The driver is called i915. You usually do not need to install anything extra.

Install Vulkan support for Intel:

sudo apt install mesa-vulkan-drivers mesa-vulkan-drivers:i386

That is all you need. Intel graphics just work.

4. Wi-Fi and Network Drivers

Most Wi-Fi chips work out of the box. Some need proprietary firmware.

Check your Wi-Fi chipset:

lspci | grep -i network

Or for USB Wi-Fi adapters:

lsusb

Common Wi-Fi chipsets and their needs:

  • Intel: Works out of the box. No extra steps.
  • Realtek: Some work, some need firmware. Check /lib/firmware for missing files.
  • Broadcom: Often needs proprietary drivers. Install:
sudo apt install firmware-b43-installer

Or for newer Broadcom chips:

sudo apt install broadcom-sta-dkms

If Wi-Fi is not working, check if firmware is missing:

dmesg | grep firmware

This shows what firmware files the kernel is looking for.

5. Audio Drivers

Audio on Linux uses ALSA (Advanced Linux Sound Architecture) and PulseAudio or PipeWire. Drivers are usually built in.

Check audio hardware:

lspci | grep -i audio

List audio devices:

cat /proc/asound/cards

If sound is not working, check if the correct card is selected:

alsamixer

Press F6 to select the correct sound card. Make sure channels are not muted (MM means muted, press M to unmute).

For USB audio devices: They usually work immediately. Check with lsusb to confirm the device is detected.

6. Printer Drivers

Printers on Linux are managed by CUPS (Common Unix Printing System). Most modern printers work with driverless printing (IPP Everywhere).

Install CUPS:

sudo apt install cups sudo systemctl enable cups sudo systemctl start cups

Access the web interface at http://localhost:631 to add printers.

For printers that need specific drivers: Many HP printers work with HPLIP:

sudo apt install hplip

For Brother printers, the manufacturer provides .deb packages on their website.

Check if your printer is detected:

lpstat -t

7. Bluetooth Drivers

Bluetooth drivers are included. You may need to install the Bluetooth stack.

Install Bluetooth support:

sudo apt install bluetooth bluez bluez-tools

Check Bluetooth controller:

lsusb | grep -i bluetooth

Or for internal Bluetooth:

lspci | grep -i bluetooth

Start Bluetooth service:

sudo systemctl enable bluetooth sudo systemctl start bluetooth

If your Bluetooth adapter is not working, check for missing firmware:

dmesg | grep -i bluetooth

8. Firmware (Non-Free Drivers)

Some hardware needs firmware files that are not included by default. These are small files that the driver loads onto the hardware at boot.

Install common firmware packages:

sudo apt install firmware-linux firmware-linux-nonfree firmware-misc-nonfree

For Wi-Fi firmware:

sudo apt install firmware-iwlwifi firmware-realtek firmware-brcm80211

For audio firmware:

sudo apt install firmware-sof-signed

After installing firmware, reboot or reload the driver:

sudo modprobe -r driver_name && sudo modprobe driver_name

9. Checking Which Drivers Are Loaded

Here are the most useful commands for seeing what drivers are active.

List all loaded kernel modules with their dependencies:

lsmod

Show driver for a specific PCI device:

lspci -k -s 00:02.0

Replace the address with your device's PCI address.

Show what driver is controlling a USB device:

lsusb -t

Show kernel messages about driver loading:

dmesg | grep -i driver

Check graphics driver details:

glxinfo -B

10. Updating Drivers

Since most drivers are in the kernel, updating your kernel updates your drivers.

Update the kernel:

sudo apt update sudo apt upgrade

This installs the latest kernel available in your distribution's repositories.

If you want a newer kernel than your distribution provides: You can use backports or a mainline kernel tool. Be careful with this on production systems.

For NVIDIA drivers, update separately:

sudo apt update sudo apt upgrade nvidia-driver-*

For Mesa graphics drivers: They update through regular system updates. On some distributions, you can use a PPA for newer Mesa:

sudo add-apt-repository ppa:kisak/kisak-mesa sudo apt update sudo apt upgrade

11. Troubleshooting Driver Issues

Hardware not working at all: First, check if it is detected:

lspci # or lsusb

If not detected, check BIOS settings. Some hardware can be disabled in BIOS.

Driver is loaded but hardware not working: Check kernel messages for errors:

dmesg | grep -i error dmesg | grep -i fail

Missing firmware errors: If dmesg shows "firmware: failed to load", find and install the package that contains that firmware:

apt search firmware | grep -i keyword

Graphics issues after update: If an update broke graphics, you can boot an older kernel from the GRUB menu (hold Shift during boot).

Module not loading at boot: You can force a module to load early by adding it to /etc/modules:

echo "module_name" | sudo tee -a /etc/modules

Blacklisting a driver: If the wrong driver is loading for your hardware, you can blacklist it. Create a file in /etc/modprobe.d:

echo "blacklist driver_name" | sudo tee /etc/modprobe.d/blacklist-driver.conf

Then update initramfs and reboot:

sudo update-initramfs -u sudo reboot

Check what drivers are available for your hardware: The modinfo command shows details about a kernel module:

modinfo i915

Reload a driver without rebooting:

sudo modprobe -r driver_name sudo modprobe driver_name

That covers driver management on Linux. Most of the time, everything just works. When it does not, the tools and commands here will help you figure out what is wrong. The kernel logs (dmesg) are your best friend when troubleshooting driver issues.
