Customizing Linux Guide

Your computer is yours. The way it looks when you boot up, the login screen, the wallpaper, the icons, the way windows behave, all of it can be changed. This guide walks through every layer of customization, from the moment you turn on your machine to the moment you close your last window. Everything here works on any system that uses these components, regardless of what distribution you are running.

1. GRUB Bootloader Theme

GRUB (Grand Unified Bootloader) is what you see when you first turn on your computer - the menu that lets you choose which operating system to boot. It can be themed with backgrounds, fonts, and custom layouts.

Theme Location and Structure

GRUB themes live in:

/boot/grub/themes/
/usr/share/grub/themes/

A GRUB theme is a directory containing:

  • theme.txt - the main configuration file
  • background.png or background.jpg - the background image
  • *.pf2 - font files (GRUB-specific format)
  • *.png - icons for menu items, progress bars, etc.

Example theme.txt structure:

title-text: "GRUB Boot Menu"
desktop-image: "background.png"
desktop-color: "#000000"

terminal-font: "DejaVuSans-12.pf2"

+ boot_menu {
  left = 30%
  top = 40%
  width = 40%
  height = 30%
  item_color = "#ffffff"
  selected_item_color = "#000000"
  item_font = "DejaVuSans-12.pf2"
  selected_item_font = "DejaVuSans-12.pf2"
  item_pixmap_style = "menu_item.png"
  selected_item_pixmap_style = "menu_item_selected.png"
}

Configuring GRUB Themes

To apply a theme, edit /etc/default/grub and add or modify:

GRUB_THEME="/boot/grub/themes/your-theme/theme.txt"
GRUB_GFXMODE="1920x1080"
GRUB_GFXPAYLOAD_LINUX="keep"

After making changes, update GRUB:

sudo update-grub

Or on some systems:

sudo grub-mkconfig -o /boot/grub/grub.cfg

Tools for GRUB Customization

Several tools can help you manage GRUB themes:

sudo apt install grub-customizer # GUI tool for GRUB configuration sudo apt install grub-theme-starfield # Install a sample theme

You can also download themes from sites like gnome-look.org. Extract them to and update GRUB.

2. Plymouth Boot Splash

Plymouth is the graphical boot splash that appears after GRUB and before the login screen. It shows the boot process with animations and progress indicators.

Available Plymouth Themes

Common Plymouth themes include:

sudo apt install plymouth-theme-* # List all available themes

Some popular themes:

  • plymouth-theme-spinfinity - rotating infinity symbol
  • plymouth-theme-solar - sun-like animation
  • plymouth-theme-breeze - KDE's Breeze theme
  • plymouth-theme-lone - minimal text-based

Installing and Switching Themes

Install a theme:

sudo apt install plymouth-theme-breeze

Switch to a different theme:

sudo update-alternatives --config default.plymouth

Select the number corresponding to your desired theme, then update the initramfs:

sudo update-initramfs -u

To preview a theme without rebooting:

sudo plymouthd --debug --tty --no-daemon sudo plymouth --show-splash sudo plymouth --quit

Creating Custom Plymouth Themes

A Plymouth theme is a directory in containing:

  • *.plymouth - the theme configuration file
  • *.png - images for the animation
  • *.script - optional script for advanced animations

To create a custom theme, copy an existing theme and modify the images and configuration.

3. SDDM Login Manager Theme

SDDM (Simple Desktop Display Manager) is the login screen. It is highly themeable and supports both Qt-based and QML-based themes.

SDDM Theme Locations

SDDM themes are stored in:

/usr/share/sddm/themes/
~/.local/share/sddm/themes/

Each theme is a directory containing:

  • theme.conf - theme configuration
  • Main.qml - QML interface file
  • *.png - background and UI images
  • fonts/ - optional fonts directory

Switching SDDM Themes

Using the system settings (graphical):

Go to System Settings > Startup and Shutdown > Login Screen (SDDM). Click "Get New SDDM Themes" to download themes, or select from installed ones.

Using the terminal:

sudo nano /etc/sddm.conf

Add or modify:

[Theme]
Current=your-theme-name

To see installed themes:

ls /usr/share/sddm/themes/

Manual SDDM Configuration

SDDM can be extensively configured. The main configuration file is . Common settings:

[Theme]
Current=breeze
CursorTheme=breeze_cursors
Font=Noto Sans,10

[Users]
DefaultSession=plasma
HideUsers=root
RememberLastUser=true

[General]
HaltCommand=/usr/bin/systemctl poweroff
RebootCommand=/usr/bin/systemctl reboot
Numlock=on

To configure a specific theme's settings, look for theme.conf inside the theme directory.

4. KSplash (Kirigami) Splash Screen

KSplash is the splash screen that appears after login while the desktop loads. KDE Plasma uses Kirigami-based splash screens.

Splash Screen Locations

KSplash themes are stored in:

/usr/share/plasma/look-and-feel/
~/.local/share/plasma/look-and-feel/

Look-and-feel packages often include splash screens. Standalone splash screens are stored in:

/usr/share/plasma/splash/
~/.local/share/plasma/splash/

Selecting and Installing Splash Screens

To change the splash screen:

System Settings > Appearance > Splash Screen. Select from the list or click "Get New Splash Screens" to download more.

To manually install a splash screen, download a .tar.gz file and extract it to . It should appear in the settings immediately.

Each splash screen directory contains:

  • metadata.desktop - theme metadata
  • contents/ - the actual QML files and images

5. Wallpapers

Wallpapers set the visual tone for your desktop environment. They can be static images, slideshows, or even dynamic wallpapers that change based on time.

System Wallpaper Directories

System-wide wallpapers are stored in:

/usr/share/wallpapers/
/usr/share/backgrounds/

User-installed wallpapers typically go in:

~/Pictures/Wallpapers/
~/.local/share/wallpapers/

Each wallpaper can be a simple image or a folder with metadata:

~/.local/share/wallpapers/MyWallpaper/
├── metadata.desktop
├── contents/
│   ├── images/
│   │   ├── 1920x1080.png
│   │   ├── 2560x1440.png
│   │   └── 3840x2160.png
│   └── screenshots/
│       └── preview.png

Setting Wallpapers

Using the graphical interface:

Right-click the desktop > Configure Desktop and Wallpaper. Choose image, slideshow, or dynamic wallpaper.

Using the terminal (Plasma-specific):

plasma-apply-wallpaperimage /path/to/image.jpg

Creating Slideshows

To create a wallpaper slideshow, create a folder with your images, then in Desktop Settings, select "Slideshow" and point it to that folder. Configure interval and transition effects.

For more advanced slideshows, you can create a slideshow.ini file:

[General]
interval=300
transition=CrossFade

[Slide 1]
file=/home/user/Pictures/wallpapers/sunset.jpg
duration=60

[Slide 2]
file=/home/user/Pictures/wallpapers/mountain.jpg
duration=60

6. Mouse Cursor Themes

Cursor themes change how your pointer looks and behaves.

Cursor Theme Locations

Cursor themes are stored in:

/usr/share/icons/
~/.local/share/icons/
~/.icons/

Each cursor theme is a directory containing:

  • cursor.theme - theme metadata
  • cursors/ - the actual cursor files (X11 format)
  • Optional index.theme for icon theme compatibility

Popular cursor themes include:

  • Breeze - KDE's default, clean and modern
  • Oxygen - the classic KDE cursor set
  • Adwaita - GNOME's cursor theme
  • Polar, Capitaine, Bibata - community favorites

Switching Cursor Themes

Using the graphical interface:

System Settings > Appearance > Cursors. Select from the list or click "Get New Cursor Themes" to download more.

Using the terminal:

sudo update-alternatives --config x-cursor-theme

Or set per-user:

nano ~/.config/kcminputrc 
[Mouse]
cursorTheme=breeze_cursors
cursorSize=24

Cursor changes take effect after logging out and back in, or by restarting the application.

7. Icon Themes

Icon themes determine how application icons, folder icons, and system icons look.

Icon Theme Locations

Icon themes are stored in:

/usr/share/icons/
~/.local/share/icons/
~/.icons/

An icon theme directory contains:

  • index.theme - theme metadata and inheritance
  • actions/, apps/, devices/, etc. - icon subdirectories
  • scalable/ - SVG icons
  • 22x22/, 48x48/, 256x256/ - raster icon sizes

Popular icon themes:

  • Breeze - KDE's default, consistent and modern
  • Papirus - colorful, covers many applications
  • Adwaita - GNOME's default
  • Numix - flat, modern design
  • Flat Remix - colorful and consistent

Switching Icon Themes

Using the graphical interface:

System Settings > Appearance > Icons. Select from the list or click "Get New Icons" to download more.

Using the terminal:

nano ~/.config/kdeglobals 
[Icons]
Theme=papirus

For Qt applications outside KDE, you may also need to set:

qt6ct & # Qt6 Configuration Tool (if installed)

Custom Icons

To override a specific icon, place a PNG or SVG file at:

~/.local/share/icons/hicolor/48x48/apps/myapp.png

Then update the icon cache:

gtk-update-icon-cache ~/.local/share/icons/hicolor

8. Qt Style (Widget Style)

Qt styles control how the widgets in Qt applications look - buttons, scrollbars, menus, and other interface elements.

Qt Style Locations

Qt styles are installed as libraries in:

/usr/lib/x86_64-linux-gnu/qt5/plugins/styles/
/usr/lib/x86_64-linux-gnu/qt6/plugins/styles/
~/.local/lib/qt5/plugins/styles/

Common built-in styles:

  • Breeze - KDE's default, modern and clean
  • Fusion - cross-platform, professional look
  • Windows - classic Windows 9x style
  • Fluent, Materia - community styles

Switching Qt Styles

Using the graphical interface:

System Settings > Appearance > Application Style. Select from the list. Click "Configure GNOME/GTK Application Style" for GTK integration.

Using the terminal (Qt5):

export QT_STYLE_OVERRIDE=Breeze

For persistent configuration:

nano ~/.config/qt5ct/qt5ct.conf 
[Appearance]
style=Breeze

For Qt6, use qt6ct similarly.

To see all available styles:

find /usr/lib -name "*.so" | grep styles

9. Kvantum Theme Engine

Kvantum is an SVG-based theme engine for Qt applications. It allows for highly detailed, customizable themes with transparency, gradients, and custom widget designs.

Installing Kvantum

sudo apt install kvantum kvantum-qt5

For Qt6 support, you may need additional packages.

Kvantum Theme Locations

Kvantum themes are stored in:

/usr/share/Kvantum/
~/.config/Kvantum/

Each theme is a directory containing:

  • *.svg - the SVG theme file
  • *.kvconfig - configuration file
  • *.qss - optional Qt Style Sheet overrides

Popular Kvantum themes:

  • KvArc - Arc-style theme
  • KvMateria - Material Design style
  • KvOxygen - Oxygen style adaptation
  • Nordic - Nord color scheme

Applying Kvantum Themes

Launch Kvantum Manager:

kvantummanager

Select a theme and click "Apply".

To use Kvantum as the default Qt style:

System Settings > Appearance > Application Style. Select "kvantum" from the list.

To configure Kvantum from the terminal:

kvantummanager --set ThemeName

For per-application overrides, you can set:

env QT_STYLE_OVERRIDE=kvantum application-name

10. Bashrc (Shell Customization)

The file is executed every time you open an interactive bash shell. It is where you customize your terminal experience.

The .bashrc File

Your file is located in your home directory. If it doesn't exist, create it:

touch ~/.bashrc

After editing, apply changes with:

source ~/.bashrc

Or open a new terminal.

Customizing the Prompt

The PS1 variable controls your prompt. A simple colorful prompt:

PS1='\[\033[01;32m\]\u@\h\[\033[00m\]:\[\033[01;34m\]\w\[\033[00m\]\$ '

Common prompt elements:

  • \u - username
  • \h - hostname
  • \w - current working directory (full path)
  • \W - basename of current directory
  • \$ - $ for normal user, # for root
  • \t - current time (24-hour)
  • \d - date
  • \n - newline

Color codes (wrap in \[\033[ and m\]):

  • 01;32m - bold green
  • 01;34m - bold blue
  • 01;31m - bold red
  • 01;33m - bold yellow
  • 00m - reset

Useful Aliases

# Navigation
alias ..='cd ..'
alias ...='cd ../..'
alias ll='ls -alF'
alias la='ls -A'
alias l='ls -CF'

# Package management shortcuts
alias update='sudo apt update && sudo apt upgrade'
alias install='sudo apt install'
alias remove='sudo apt remove'
alias search='apt search'

# System
alias cls='clear'
alias ports='netstat -tulanp'
alias meminfo='free -m -l -t'
alias cpuinfo='lscpu'

# Git shortcuts
alias gs='git status'
alias ga='git add'
alias gc='git commit -m'
alias gp='git push'
alias gl='git log --oneline --graph'

# Safety
alias rm='rm -i'
alias cp='cp -i'
alias mv='mv -i'

Custom Functions

# Create a directory and cd into it
mkcd() {
    mkdir -p "$1" && cd "$1"
}

# Extract any compressed file
extract() {
    if [ -f $1 ] ; then
        case $1 in
            *.tar.bz2)   tar xjf $1     ;;
            *.tar.gz)    tar xzf $1     ;;
            *.bz2)       bunzip2 $1     ;;
            *.rar)       unrar x $1     ;;
            *.gz)        gunzip $1      ;;
            *.tar)       tar xf $1      ;;
            *.tbz2)      tar xjf $1     ;;
            *.tgz)       tar xzf $1     ;;
            *.zip)       unzip $1       ;;
            *.Z)         uncompress $1  ;;
            *.7z)        7z x $1        ;;
            *)           echo "'$1' cannot be extracted" ;;
        esac
    else
        echo "'$1' is not a valid file"
    fi
}

# Find file by name
ff() {
    find . -type f -iname "*$1*" 2>/dev/null
}

# Kill process by name
killp() {
    pkill -f "$1"
}

11. KDE Color Schemes

Color schemes control the colors used throughout KDE Plasma applications - window backgrounds, text colors, selection highlights, and more.

Color Scheme Locations

Color schemes are stored as .colors files in:

/usr/share/color-schemes/
~/.local/share/color-schemes/

A color scheme file looks like:

[ColorEffects:Disabled]
Effect=1

[ColorEffects:Inactive]
ChangeSelectionColor=1
Effect=1

[Colors:Window]
BackgroundNormal=43,45,48
BackgroundAlternate=53,55,58
ForegroundNormal=248,248,242
ForegroundInactive=127,127,127
ForegroundActive=248,248,242

[Colors:Selection]
BackgroundNormal=61,174,233
ForegroundNormal=255,255,255

[Colors:Button]
BackgroundNormal=49,51,54
ForegroundNormal=248,248,242

[Colors:View]
BackgroundNormal=35,37,39
ForegroundNormal=248,248,242

[Colors:Tooltip]
BackgroundNormal=49,51,54
ForegroundNormal=248,248,242

[General]
Name=MyCustomTheme
Description=A custom color scheme

Applying Color Schemes

Using the graphical interface:

System Settings > Appearance > Colors. Select from the list or click "Get New Color Schemes" to download more.

To set from the terminal:

plasma-apply-colorscheme "MyCustomTheme"

Or edit :

[General]
ColorScheme=MyCustomTheme

Creating Custom Color Schemes

To create a custom color scheme:

  1. Go to System Settings > Appearance > Colors
  2. Select a base scheme
  3. Click "Edit" to open the color editor
  4. Modify colors for each element
  5. Click "Save As" to save your custom scheme

Color schemes can also be created manually. The easiest way is to copy an existing .colors file and modify the RGB values.

Popular color schemes:

  • Breeze - KDE's default
  • Breeze Dark - dark version of Breeze
  • Nord - arctic, blueish theme
  • Dracula - dark with vibrant accents
  • Catppuccin - pastel colors in various flavors

12. KDE Aurorae Window Decorations

Aurorae is KDE's window decoration engine that uses SVG themes for window borders, title bars, and buttons. It allows for highly customizable window decorations.

Aurorae Theme Locations

Aurorae themes are stored in:

/usr/share/kwin/aurorae/
~/.local/share/kwin/aurorae/

Each theme is a directory containing:

  • metadata.desktop - theme metadata
  • *.svg - the SVG theme file
  • *.rc - optional configuration
  • preview.png - thumbnail preview

Applying Aurorae Themes

Using the graphical interface:

System Settings > Appearance > Window Decorations. Select "Aurorae" from the theme list, then click "Get New Window Decorations" to download more, or select from installed ones.

To apply from the terminal:

kwriteconfig5 --file kwinrc --group org.kde.kdecoration2 --key theme "Aurorae" kwriteconfig5 --file kwinrc --group Aurorae --key theme "ThemeName" kwin_x11 --replace & # Or kwin_wayland --replace for Wayland

Creating Aurorae Themes

Aurorae themes are SVG files with specific naming conventions for different elements:

  • titlebar.svg - the main title bar
  • close.svg, maximize.svg, minimize.svg - window buttons
  • decoration.svg - overall decoration

SVG elements use special IDs:

#titlebar
#close
#maximize
#minimize
#restore
#help
#shade
#keepabove
#keepbelow

Each button can have different states:

  • :active - normal state
  • :hover - mouse over
  • :pressed - button clicked

A simple example metadata.desktop:

[Desktop Entry]
Name=My Theme
Type=Service
X-KDE-ServiceTypes=KWin/Aurorae
X-KDE-PluginInfo-Name=mytheme
X-KDE-PluginInfo-Category=aurorae

13. Bringing It All Together

Customization is most satisfying when all elements work together. Here is a workflow for creating a cohesive look:

  1. Start with a color scheme. Choose or create colors for windows, text, and highlights. This will influence everything else.
  2. Find or create a matching window decoration. Use Aurorae to make title bars match your color scheme.
  3. Pick an icon theme. Look for one that complements your color palette.
  4. Choose a cursor theme. Should be readable against your backgrounds.
  5. Select a Qt style. Kvantum offers the most flexibility for matching your color scheme.
  6. Set a wallpaper. Choose something that works with your color scheme without overwhelming it.
  7. Theme GRUB, Plymouth, and SDDM. These are the first things users see, so they set the tone.
  8. Customize your terminal. Use bashrc to match your color scheme in the terminal as well.

Many cohesive theme packs bundle multiple components together. Look for "Global Themes" in System Settings - they can apply color schemes, icon themes, window decorations, and wallpapers all at once.

Your customizations live in and . Back these up if you want to preserve your setup across reinstalls.

Customization is about making your computer feel like yours. Every setting here can be changed, tweaked, or completely replaced. Start with one component, get comfortable, then move to the next. The system is designed to be flexible - take advantage of it.
