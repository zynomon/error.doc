Setting Up a Server

So you want to set up a server. Whether it is for hosting websites, storing files, running game servers, or just learning how things work, this guide walks you through the basics and covers different types of servers you can run.

1. Before You Start

Before setting up a server, make sure you have:

  • A working system with root access or sudo privileges
  • A stable internet connection
  • A static IP address or domain name if the server needs to be accessible from outside your network
  • Firewall rules configured on your router if you are hosting services to the internet
  • At least 1GB RAM for basic servers, more for game servers or databases

You can set up a server on a spare computer, a virtual machine, or a cloud instance. The commands in this guide work the same way on any modern Linux system.

2. Base Server Installation

Start with a minimal installation of your operating system. The netinstall ISO with "SSH server" selected during installation gives you a clean base.

Once installed, log in and update everything:

sudo apt update sudo apt upgrade -y

Install essential tools:

sudo apt install -y curl wget git htop neofetch ufw fail2ban

Set your hostname (replace yourserver with what you want):

sudo hostnamectl set-hostname yourserver

Make sure the system clock is correct:

sudo timedatectl set-timezone Your/Timezone sudo timedatectl set-ntp yes

Check it with timedatectl.

3. Security Basics

A few simple steps make your server much safer.

Set up firewall with UFW:

sudo ufw allow ssh sudo ufw enable

If you are running web services later, add sudo ufw allow 80 and sudo ufw allow 443.

Disable root login over SSH: Edit

sudo nano /etc/ssh/sshd_config

Find and set:

PermitRootLogin no
PasswordAuthentication no

But only disable password authentication if you have SSH keys set up. Otherwise, leave it as yes and use strong passwords.

Reload SSH:

sudo systemctl reload sshd

Install fail2ban to block brute force attacks:

sudo apt install fail2ban -y sudo systemctl enable fail2ban sudo systemctl start fail2ban

Set up automatic security updates:

sudo apt install unattended-upgrades -y sudo dpkg-reconfigure --priority=low unattended-upgrades

Select Yes when prompted.

4. Server Types

Here are the most common server setups and how to get them running.

Web Server (Apache / Nginx)

A web server serves websites and web applications. Nginx is lighter and faster for static content and reverse proxying. Apache is more flexible with .htaccess files and module configuration.

Install Nginx:

sudo apt install nginx -y sudo systemctl enable nginx sudo systemctl start nginx

Your web root is at . Test by visiting your server's IP address in a browser. You should see the default Nginx page.

Install Apache:

sudo apt install apache2 -y sudo systemctl enable apache2 sudo systemctl start apache2

Apache's web root is at .

Add PHP support (for dynamic sites like WordPress):

sudo apt install php-fpm php-mysql -y

Then configure your web server to use PHP-FPM. The exact configuration varies, but most guides cover this step.

Database Server (MySQL / MariaDB / PostgreSQL)

Databases store data for web applications, game servers, and other services.

Install MariaDB (the open-source MySQL replacement):

sudo apt install mariadb-server -y sudo systemctl enable mariadb sudo systemctl start mariadb sudo mysql_secure_installation

The secure installation script asks you to set a root password, remove anonymous users, and disable remote root login. Answer yes to all.

Install PostgreSQL:

sudo apt install postgresql postgresql-contrib -y sudo systemctl enable postgresql sudo systemctl start postgresql

PostgreSQL uses a different authentication model. Switch to the postgres user to manage it:

sudo -u postgres psql

File Server (Samba / NFS)

File servers let other computers on your network access shared folders.

Samba (Windows and Linux sharing):

sudo apt install samba -y

Edit the configuration at :

sudo nano /etc/samba/smb.conf

Add a share at the bottom:

[myshare]
   path = /srv/samba/myshare
   browseable = yes
   read only = no
   guest ok = no
   create mask = 0755

Create the directory and set permissions:

sudo mkdir -p /srv/samba/myshare sudo chmod 755 /srv/samba/myshare sudo smbpasswd -a yourusername

Restart Samba:

sudo systemctl restart smbd

NFS (Linux to Linux sharing):

sudo apt install nfs-kernel-server -y

Edit exports file at :

sudo nano /etc/exports

Add:

/srv/nfs 192.168.1.0/24(rw,sync,no_subtree_check)

Create the directory and restart:

sudo mkdir -p /srv/nfs sudo systemctl restart nfs-server

Game Server (Minecraft / SteamCMD)

Hosting your own game server gives you full control over settings, mods, and who can join.

Minecraft Java Edition Server:

sudo apt install openjdk-17-jre-headless -y mkdir ~/minecraft cd ~/minecraft wget https://piston-data.mojang.com/v1/objects/.../server.jar java -Xmx1024M -Xms1024M -jar server.jar nogui

The first run creates . Edit it to accept the agreement:

nano ~/minecraft/eula.txt

Change eula=false to eula=true. Run the server again.

SteamCMD (for games like Counter-Strike, Rust, ARK):

sudo apt install steamcmd -y mkdir ~/gameserver cd ~/gameserver steamcmd +login anonymous +force_install_dir ~/gameserver +app_update 740 +quit

Replace 740 with the app ID of the game you want to host. You can find app IDs on the Valve Developer Community wiki.

Media Server (Jellyfin / Plex)

Media servers organize your movies, TV shows, and music and let you stream them to other devices.

Jellyfin (open-source):

sudo apt install curl -y curl https://repo.jellyfin.org/install-debuntu.sh | sudo bash sudo systemctl enable jellyfin sudo systemctl start jellyfin

Access the web interface at http://yourserverip:8096 and follow the setup wizard.

Plex:

wget https://downloads.plex.tv/plex-media-server-new/.../plexmediaserver_*.deb sudo dpkg -i plexmediaserver_*.deb sudo systemctl enable plexmediaserver sudo systemctl start plexmediaserver

Access Plex at http://yourserverip:32400/web.

Docker Host

Docker lets you run applications in isolated containers. It is great for running multiple services without conflicts.

sudo apt install docker.io docker-compose -y sudo systemctl enable docker sudo systemctl start docker sudo usermod -aG docker $USER

Log out and back in for the group changes to take effect. Test with:

docker run hello-world

Mail Server (Basic)

Running a mail server is complex and needs careful setup to avoid being marked as spam. This is a basic setup for learning or internal use.

sudo apt install postfix dovecot-core dovecot-imapd -y

During installation, select "Internet Site" and set your domain. Postfix configuration files are at . Dovecot config is at .

This is a minimal setup. For a production mail server, you also need SPF, DKIM, and DMARC records configured in your DNS.

5. How to Choose What You Need

Not sure which server type to set up? Here is a quick guide:

Start with a web server if: You want to host a personal website, blog, or web application. Nginx is easier for beginners and handles static files well.

Add a database if: Your website needs to store user data, posts, or content. MariaDB is the simplest choice.

Set up a file server if: You have multiple computers at home and want to share files between them. Samba works with Windows machines too.

Try a game server if: You play multiplayer games with friends and want full control over the server settings.

Install a media server if: You have a collection of movies or music and want to stream them to your TV, phone, or other devices.

Use Docker if: You plan to run multiple services and want an easy way to manage them without worrying about dependencies.

Skip mail server unless: You are doing it for learning or have specific internal needs. Email hosting is complex and most people are better off using a dedicated email service.

6. Ongoing Maintenance

A server needs regular care to stay secure and run smoothly.

Check logs regularly: Logs live in . The main ones to watch are and .

sudo tail -f /var/log/syslog

Monitor disk space:

df -h

If disk space gets low, use ncdu to find large files:

sudo apt install ncdu -y sudo ncdu /

Update regularly: Set a reminder to run updates weekly:

sudo apt update && sudo apt upgrade -y

Reboot when needed: Kernel updates require a reboot. Check if a reboot is pending:

ls /var/run/reboot-required

If that file exists, reboot when convenient:

sudo reboot

Back up important data: At minimum, back up (configurations) and any data directories you care about. A simple script with rsync to another machine or external drive works well.

7. Troubleshooting Common Issues

Cannot connect to the server over SSH: Check if the service is running:

sudo systemctl status ssh

Check firewall rules:

sudo ufw status

Make sure port 22 is allowed.

Web server not showing pages: Check if the service is running:

sudo systemctl status nginx # or sudo systemctl status apache2

Check firewall:

sudo ufw status

Ports 80 and 443 should be allowed. Check or for specific errors.

Game server not showing up: Make sure the port is open in the firewall and forwarded on your router. Most game servers need UDP ports forwarded. Check the server logs in the installation directory.

Service fails to start: Look at the logs:

sudo journalctl -u servicename -f

Replace servicename with the actual service name like nginx, mysql, or jellyfin.

Server feels slow: Check system load:

htop free -h df -h

High CPU, low memory, or full disk can all cause slowdowns.

Cannot write to shared folder on file server: Check permissions on the shared directory:

ls -la /path/to/share

Make sure the user has write permissions. For Samba, also check the smb.conf share settings.
