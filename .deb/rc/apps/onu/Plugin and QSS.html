Onu Browser Plugins and QSS Themes Guide

Onu supports two main ways to extend and customize the browser: plugins (dynamic extensions) and QSS themes (style sheets). This guide explains how both systems work, how to create your own, and how to use them effectively.

1. Plugin Architecture

Onu plugins are dynamic libraries (.so on Linux, .dll on Windows) that implement the IExtension interface. They are discovered automatically from ~/.config/Onu/Conf/extensions and can be enabled or disabled through the Extension Manager.

IExtension Interface

Every plugin must implement the IExtension class. Here is the interface definition:

class IExtension {
public:
    virtual ~IExtension() {}
    virtual ExtensionMetadata metadata() const = 0;
    virtual bool apply(QMainWindow* window, Browser_Backend* backend) = 0;
    virtual QString injectScript() const { return QString(); }
    virtual void onPageCreated(QWebEnginePage* page) { Q_UNUSED(page); }
    virtual QList<QAction*> getContextMenuActions(QWebEngineView* view) const {
        Q_UNUSED(view);
        return QList<QAction*>();
    }
};

Required methods:

  • metadata() - Returns information about the extension (name, version, maintainer, description, icon)
  • apply() - Called when the extension is enabled. Use this to add toolbars, menu items, or set up functionality

Optional methods:

  • injectScript() - Returns JavaScript code to inject into every webpage
  • onPageCreated() - Called when a new web page is created. Useful for setting up page-specific hooks
  • getContextMenuActions() - Returns custom actions to add to the right-click context menu

Extension Metadata

The ExtensionMetadata struct holds basic information about your plugin:

struct ExtensionMetadata {
    QString name;           // Display name
    QString version;        // Version string
    QString maintainer;     // Author or maintainer name
    QString description;    // Short description
    QIcon icon;             // Icon shown in extension manager
};

This metadata appears in the Extension Manager and helps users identify your plugin.

Extension Lifecycle

Here is how plugins are managed:

  1. Discovery: Onu scans ~/.config/Onu/Conf/extensions for .so/.dll/.dylib files on startup
  2. Loading: Each library is loaded with QPluginLoader and checked for the IExtension interface
  3. State loading: The extension's enabled/disabled state is read from encrypted.conf
  4. Application: If enabled, apply() is called during browser startup
  5. Page interaction: For each new page, onPageCreated() is called and injectScript() is executed
  6. Context menus: When right-clicking, getContextMenuActions() is called to add custom actions

2. Creating a Plugin

Basic Structure

A minimal plugin looks like this:

#include <QObject>
#include <QtPlugin>
#include <QMainWindow>
#include <QAction>
#include "extense.H"

class MyExtension : public QObject, public IExtension {
    Q_OBJECT
    Q_PLUGIN_METADATA(IID IExtension_iid)
    Q_INTERFACES(IExtension)

public:
    ExtensionMetadata metadata() const override {
        ExtensionMetadata md;
        md.name = "My Extension";
        md.version = "1.0";
        md.maintainer = "Your Name";
        md.description = "Does something useful";
        md.icon = QIcon::fromTheme("applications-graphics");
        return md;
    }

    bool apply(QMainWindow* window, Browser_Backend* backend) override {
        Q_UNUSED(backend);

        QToolBar* toolbar = window->findChild<QToolBar*>("NavigationToolbar");
        if (toolbar) {
            QAction* action = new QAction("My Action", this);
            connect(action, &QAction::triggered, [window]() {
                // Do something
            });
            toolbar->addAction(action);
        }
        return true;
    }
};

#include "main.moc"

CMake Configuration

Here is a CMakeLists.txt that builds a plugin:

cmake_minimum_required(VERSION 3.16)
project(MyExtension LANGUAGES CXX)

set(CMAKE_AUTOUIC ON)
set(CMAKE_AUTOMOC ON)
set(CMAKE_AUTORCC ON)
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

find_package(Qt6 REQUIRED COMPONENTS Core Widgets)

# If you need the Onu headers, add the path to your Onu source
target_include_directories(MyExtension PRIVATE
    /path/to/onu/source
)

add_library(MyExtension SHARED
    main.cxx
)

target_link_libraries(MyExtension
    Qt6::Core
    Qt6::Gui
    Qt6::Widgets
)

set_target_properties(MyExtension PROPERTIES
    PREFIX ""
    LIBRARY_OUTPUT_DIRECTORY ${CMAKE_CURRENT_SOURCE_DIR}
)

After building, copy the resulting .so/.dll file to ~/.config/Onu/Conf/extensions.

Full Example: QR Code Generator

The QR Code Generator plugin demonstrates many plugin capabilities:

  • Adding a toolbar button with shortcut (Ctrl+Shift+Q)
  • Adding context menu items for pages
  • Creating complex dialogs with settings
  • Generating and saving images (PNG and SVG)
  • Using third-party libraries (libqrencode)
  • Real-time preview with sliders and color pickers

Key features shown:

// Adding a toolbar button with shortcut
QAction* action = new QAction(
    QIcon::fromTheme("qrscanner-symbolic"),
    "QR Code",
    this
);
action->setShortcut(QKeySequence("Ctrl+Shift+Q"));
connect(action, &QAction::triggered, this, [this]() {
    generateQRFromCurrentPage();
});
navToolbar->addAction(action);

// Adding context menu items
QList<QAction*> getContextMenuActions(QWebEngineView* view) const override {
    QList<QAction*> actions;
    QAction* qrAction = new QAction("Generate QR Code", view);
    connect(qrAction, &QAction::triggered, this, [this, urlStr, title]() {
        const_cast<QR_Creator*>(this)->generateQR(urlStr, title);
    });
    actions.append(qrAction);
    return actions;
}

// Getting the current active web view
QWebEngineView* getCurrentWebView() {
    if (!m_window) return nullptr;
    QWidget* central = m_window->centralWidget();
    QList<QWebEngineView*> views = central->findChildren<QWebEngineView*>();
    auto it = std::find_if(views.begin(), views.end(), [](QWebEngineView* view) {
        return view && view->isVisible();
    });
    return it != views.end() ? *it : nullptr;
}

3. Plugin API Reference

Available Classes

Plugins have access to these Onu classes (when included from the Onu source):

  • Browser_Backend - Manage favorites, history, sessions, and tabs
  • Settings_Backend - Read and write settings (public and encrypted)
  • Icon_Man - Save and load icons from cache
  • Dialogs - Open built-in dialogs (favorites, history, settings, etc.)
  • Onu_Window - The main browser window
  • Onu_Web - Web page class with extended functionality
  • Create_Toolbars - Access to toolbar widgets

To use these, include the appropriate headers and link against the Onu libraries.

What Plugins Can Do

  • Add toolbar buttons - Add actions to any toolbar (Navigation, Favorites, etc.)
  • Add menu items - Add actions to the main menu or context menus
  • Inject JavaScript - Run custom scripts on every page load
  • Modify page behavior - Connect to web engine signals and events
  • Access settings - Read and write browser settings (public or encrypted)
  • Manage favorites/history - Add, remove, or modify favorites and history entries
  • Create dialogs - Open custom windows with Qt widgets
  • Intercept network requests - Through the request interceptor system
  • Add keyboard shortcuts - Register global shortcuts
  • Use third-party libraries - Link against any library your plugin needs

Best Practices

  • Check for nullptr: Always verify that window pointers and toolbars exist before using them
  • Clean up connections: Use Qt's parent-child ownership model to avoid memory leaks
  • Use signals and slots: Connect to existing signals rather than overriding behavior when possible
  • Respect user settings: Check if features are enabled before applying changes
  • Be thread-safe: Remember that Qt's UI must be accessed from the main thread only
  • Handle missing resources: If your plugin requires external files, check for their existence gracefully

4. QSS Themes

What is QSS

QSS (Qt Style Sheets) is a CSS-like language that controls the appearance of Qt applications. Onu uses QSS to theme the entire browser interface. You can create custom themes by writing .qss files and placing them in ~/.config/Onu/Conf/themes.

Onu ships with two built-in themes: Onu-Dark and Onu-Light. You can use these as starting points for your own themes.

Theme Structure

A theme file is a plain text .qss file. Here is the basic structure:

/* Onu-Dark.qss - Example dark theme */

/* Global styles */
QWidget {
    background-color: #1e1e1e;
    color: #d4d4d4;
    selection-background-color: #264f78;
}

/* Main window */
QMainWindow {
    background-color: #1e1e1e;
}

/* Toolbars */
QToolBar {
    background-color: #2d2d2d;
    border: none;
    spacing: 3px;
    padding: 2px;
}

QToolBar::handle {
    image: url(:/images/handle.png);
}

/* Tab bar */
QTabBar::tab {
    background-color: #2d2d2d;
    color: #d4d4d4;
    padding: 6px 12px;
    margin: 2px;
    border-top-left-radius: 4px;
    border-top-right-radius: 4px;
}

QTabBar::tab:selected {
    background-color: #3c3c3c;
    border-bottom: 2px solid #007acc;
}

QTabBar::tab:hover {
    background-color: #3c3c3c;
}

/* Address bar */
QLineEdit {
    background-color: #3c3c3c;
    border: 1px solid #4c4c4c;
    border-radius: 4px;
    padding: 4px 8px;
    selection-background-color: #007acc;
}

QLineEdit:focus {
    border-color: #007acc;
}

/* Buttons */
QPushButton {
    background-color: #3c3c3c;
    border: 1px solid #4c4c4c;
    border-radius: 4px;
    padding: 4px 8px;
    color: #d4d4d4;
}

QPushButton:hover {
    background-color: #4c4c4c;
}

QPushButton:pressed {
    background-color: #2d2d2d;
}

/* Scroll bars */
QScrollBar:vertical {
    background-color: #2d2d2d;
    width: 12px;
    margin: 0px;
}

QScrollBar::handle:vertical {
    background-color: #4c4c4c;
    border-radius: 6px;
    min-height: 20px;
}

QScrollBar::handle:vertical:hover {
    background-color: #5c5c5c;
}

/* Menu bar */
QMenuBar {
    background-color: #2d2d2d;
    color: #d4d4d4;
}

QMenuBar::item:selected {
    background-color: #3c3c3c;
}

QMenu {
    background-color: #2d2d2d;
    border: 1px solid #4c4c4c;
}

QMenu::item:selected {
    background-color: #007acc;
}

Writing Themes

Finding widget names: To style specific widgets, you need to know their object names. Onu uses these object names:

  • NavigationToolbar - The main navigation toolbar
  • SearchToolbar - The address bar toolbar
  • FavoritesToolbar - The favorites sidebar toolbar
  • RecentToolbar - The recent history toolbar
  • TabToolbar - The tab bar toolbar

You can style them with selectors like:

QToolBar#NavigationToolbar {
    background-color: #2d2d2d;
}

Common selectors:

/* Type selector */
QPushButton { ... }

/* Class selector (all widgets of a class) */
.QToolButton { ... }

/* ID selector (specific object) */
QLineEdit#addressBar { ... }

/* Pseudo-states */
QPushButton:hover { ... }
QPushButton:pressed { ... }
QPushButton:checked { ... }
QPushButton:disabled { ... }

/* Child widgets */
QTabBar::tab { ... }
QScrollBar::handle { ... }
QMenu::item { ... }

Useful Selectors

Here are the main widgets you can style in Onu:

Widget Selector Notes
Main window QMainWindow Background color for the entire window
Central area QWidget#qt_centralwidget Area containing web views
Toolbars QToolBar, QToolBar#NavigationToolbar Background, borders, spacing
Tab bar QTabBar::tab, QTabBar::tab:selected Tab appearance, selected/hover states
Address bar QLineEdit Background, border, focus state
Search engine combo QComboBox Dropdown style
Buttons QPushButton, QToolButton Background, hover, pressed states
Menu bar QMenuBar, QMenuBar::item Menu bar background and item styling
Menu items QMenu, QMenu::item Dropdown menu appearance
Scroll bars QScrollBar:vertical, QScrollBar::handle Scroll bar width, handle color
Find bar QWidget#findBar The inline find widget
Dialogs QDialog, QMessageBox Settings dialog, history dialog, etc.
Status bar QStatusBar Bottom status area

Pseudo-states you can use:

  • :hover - Mouse over the widget
  • :pressed - Button is being pressed
  • :checked - Checkable button is checked
  • :disabled - Widget is disabled
  • :focus - Widget has keyboard focus
  • :selected - Item is selected (for tabs, menu items)
  • :active - Window is active
  • :inactive - Window is inactive

5. Troubleshooting

Plugin not appearing in Extension Manager:

  • Check that the .so/.dll file is in ~/.config/Onu/Conf/extensions
  • Verify the file has execute permissions
  • Check that all required Qt libraries are available
  • Look for error messages in the terminal when starting Onu

Plugin fails to load:

  • Use ldd (Linux) or Dependency Walker (Windows) to check for missing dependencies
  • Ensure the plugin was compiled with the same Qt version as Onu
  • Check that the Q_PLUGIN_METADATA macro uses the correct IID (IExtension_iid)

Theme not showing in settings:

  • Verify the .qss file is in ~/.config/Onu/Conf/themes
  • Check the file has a .qss extension
  • Restart Onu or click Refresh in the Theme Editor
  • Check for syntax errors in the QSS file (invalid properties are silently ignored)

Theme not applying correctly:

  • Use Qt's built-in style checker: qss-validator or online tools
  • Test small changes first to isolate the problem
  • Remember that QSS is case-sensitive for property names
  • Check that selectors match actual widget names

Plugin causes crashes:

  • Run Onu from terminal to see crash messages
  • Check for null pointer dereferences (especially m_window or backend)
  • Ensure connections are properly disconnected when the plugin is disabled
  • Use QPointer for widget pointers that may be deleted
