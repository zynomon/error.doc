1. Overview

The Onu downloader consists of two main components:

  • Qtdl: The core download engine that handles HTTP requests, segmented downloads, and network operations
  • DL_Man: The user interface that displays downloads, manages the download list, and handles user interactions

Key Features

  • Multi-threaded segmented downloads: Splits large files into chunks for faster downloads (1-32 segments)
  • Pause and resume: Pause downloads at any time and resume later, even after restarting the browser
  • Smart path organization: Automatically organizes downloads into folders based on file type (Images, Documents, Videos, Music, Archives, Others)
  • Automatic duplicate handling: Adds numbers to filenames when duplicates exist
  • Session persistence: Saves download states to disk so downloads survive browser restarts
  • HTTP range request support: Automatically detects if servers support range requests for segmented downloads
  • File watcher integration: Monitors downloaded files and updates UI if files are moved or deleted externally

Architecture Overview

The downloader runs in its own thread to avoid blocking the UI. Here is the basic flow:

+----------------+     +----------------+     +----------------+
|   Browser      |     |   DL_Man       |     |    Qtdl        |
|   (Web Page)   |---->|   (UI)         |---->|   (Engine)     |
+----------------+     +----------------+     +----------------+
                              |                       |
                              |                       |
                              v                       v
                       +----------------+     +----------------+
                       |  Download      |     |  HTTP Network  |
                       |  Index File    |     |  Requests      |
                       +----------------+     +----------------+

When a download starts:

  1. The browser calls DL_Man::addDownload()
  2. DL_Man creates a Qtdl instance and moves it to a worker thread
  3. Qtdl sends a HEAD request to check server capabilities (range support, content length)
  4. If range requests are supported and the file is large enough, segmented download begins
  5. Otherwise, a single-threaded download starts
  6. Progress updates are sent back to DL_Man via signals
  7. When finished, the file is merged (if segmented) and moved to the final location

2. Download States

Each download can be in one of five states:

State Description User Action Available
Queued Waiting to start (not currently used in implementation) None
Downloading Actively downloading data Pause, Cancel
Paused Download suspended, segment files cached on disk Resume, Cancel
Finished Download completed successfully Open, Show in Folder, Remove
Cancelled_Failed Download was cancelled by user or failed due to network error Retry, Remove

State transitions are handled automatically by the downloader and reflected in the UI with appropriate buttons and status messages.

3. Segmented Downloads

How Segmented Downloads Work

When a server supports HTTP range requests (Accept-Ranges: bytes header), Qtdl can split the file into multiple segments and download them in parallel. Here is how it works:

  1. The server is queried for total file size via a HEAD request
  2. The file is divided into N segments (configurable, default 8)
  3. Each segment is downloaded independently to a temporary .part file
  4. Each segment request includes a Range header: Range: bytes=start-end
  5. Segments are downloaded in parallel using separate QNetworkAccessManager instances
  6. When all segments complete, they are merged into the final file in order
  7. Segment files are deleted after successful merge

Segment storage: Segments are stored in ~/Downloads/.dlcache with filenames like filename.part0, filename.part1, etc.

Segment Configuration

The number of segments can be configured in the Downloads Manager window. Options range from 1 to 32 segments. This setting applies to new downloads only.

Considerations when choosing segment count:

  • More segments = faster downloads on high-bandwidth connections with servers that support range requests
  • Too many segments can overwhelm some servers or cause connection limits to be exceeded
  • For small files (<10MB), 1-2 segments is usually sufficient
  • For large files (100MB+), 8-16 segments can significantly improve speed

When Segmented Downloads Are Used

Segmented downloads are automatically enabled when ALL of these conditions are met:

  • The server responds with Accept-Ranges: bytes header
  • The Content-Length header is present (file size is known)
  • File size is greater than 0
  • Number of segments is greater than 1
  • Transfer-Encoding is not "chunked"

If any condition fails, the downloader falls back to single-threaded mode.

4. Smart Path Organization

When Smart Path is enabled in Settings, downloaded files are automatically organized into subfolders based on file extension.

File Type Folders

File Type Extensions Destination Folder
Documents .pdf, .doc, .docx, .txt, .odt Documents/
Images .jpg, .jpeg, .png, .gif, .bmp, .svg Images/
Videos .mp4, .avi, .mkv, .mov, .wmv, .flv Videos/
Music .mp3, .wav, .flac, .ogg, .m4a Music/
Archives .zip, .rar, .7z, .tar, .gz Archives/
Others Any other extension Others/

Folders are created automatically if they don't exist.

Duplicate File Handling

When a file with the same name already exists in the destination folder, the downloader automatically appends a number:

  • First duplicate: filename (1).ext
  • Second duplicate: filename (2).ext
  • And so on, up to 999 duplicates

This happens before the download starts, so you never get a prompt asking what to do with existing files.

5. Downloads Manager Window

The Downloads Manager (accessible via Ctrl+J or from the menu) provides a comprehensive view of all downloads.

Download List Interface

Each download item displays:

  • File icon: Based on file type from system icon theme
  • Filename: The name of the downloaded file (strikethrough if file is missing)
  • Age: Time since download completed (Today, Yesterday, etc.)
  • Status: Download progress or final status
  • Progress bar: Visual representation of download progress
  • Action button: Context-dependent (Pause, Resume, Retry, Remove)
  • Open button: Opens the downloaded file (visible for completed downloads)

Context Menu Options

Right-click any download item to access:

  • Open: Open the downloaded file (completed downloads only)
  • Show in Folder: Open the containing folder in file manager
  • Pause/Resume/Retry: Control active downloads
  • Cancel: Cancel an active or paused download
  • Remove: Delete the download entry and optionally delete the file

Grouping by Date

Downloads are automatically grouped into sections:

  • Today: Downloads from current calendar day
  • Yesterday: Downloads from previous day
  • This Week: Downloads from the last 7 days (excluding Today/Yesterday)
  • Older: Downloads older than 7 days

These groups update automatically when downloads are added or completed.

Clear Completed button: Deletes all finished downloads from the list AND deletes the actual files from disk.

Open Download Folder button: Opens the main download directory in your file manager.

Segments spin box: Configures the number of segments for new downloads (1-32).

6. Persistence and Session Recovery

Download Index File

Download states are saved to ~/Downloads/.dlcache/.onudl. This file uses a simple INI-like format:

[global]
segments=8

[finished]
1=https://example.com/file.pdf|Documents/example.pdf
2=https://example.com/image.png|Images/image.png

[cancelled_failed]
1=https://example.com/failed.zip

[paused]
1=https://example.com/largefile.iso|.dlcache/largefile.part0,.dlcache/largefile.part1

When Onu restarts, it reads this file and restores downloads that were in progress, paused, or completed. Completed downloads that no longer exist on disk are moved to the Cancelled/Failed section.

State Persistence

The following states are preserved across browser sessions:

  • Segment count preference
  • Finished downloads (file path stored, verified on next launch)
  • Paused downloads (segment files preserved)
  • Cancelled/failed downloads (for reference and retry)

Active downloads (Downloading state) are automatically cancelled when the browser closes. This is intentional to prevent corruption and simplify state management.

7. API Reference

Qtdl Class

The Qtdl class is the core download engine. It runs in its own thread and handles all network operations.

Public Methods:

// Constructor: numSegments = number of parallel segments (1-32)
explicit Qtdl(int numSegments = 8);

// Start a download
void startDownload(const QUrl &url, const QString &finalPath, const QString &cacheDir);

// Control an active download
void pauseDownload();
void resumeDownload();
void cancelDownload();

// Query state
bool isDownloading() const;
bool isCancelled() const;
bool isPaused() const;
bool isUnknownLength() const;  // true if server didn't send Content-Length

// Get segment file paths (for persistence)
QStringList getSegmentPaths() const;

Signals:

void progress(qint64 bytesReceived, qint64 bytesTotal, double speedKBps);
void downloadFinished(const QString &finalPath, bool success, const QString &error);
void downloadPaused(const QString &finalPath);
void downloadResumed(const QString &finalPath);

Internal Workflow:

  1. doDownload() initializes state and calls checkServerCapabilities()
  2. checkServerCapabilities() sends HEAD request to detect range support
  3. If range supported, startSegmentedDownload() creates segments
  4. Otherwise, startSingleDownload() uses standard download
  5. Each segment runs in its own network manager and writes to a .part file
  6. When all segments complete, mergeSegments() combines them
  7. Segment files are deleted on successful merge

DL_Man Class

DL_Man inherits from QDialog and provides the downloads manager UI.

Public Methods:

// Add a new download to the manager
void addDownload(const QUrl &url, const QString &fileName);

// Save all download states (called on exit)
void saveAllStatesOnExit();

Signals:

void downloadAdded();  // Emitted when a new download is added

Internal Components:

  • DownloadItem struct: Holds all data for a single download (downloader pointer, UI widgets, state, file info)
  • m_downloads: Vector of all download items
  • m_fileWatcher: Monitors downloaded files for external changes
  • saveIndex() / loadIndex(): Persistence to .onudl file

UI Components:

  • QListWidget: Main download list with custom widgets per item
  • QStackedWidget: Switches between download list and empty state view
  • QSpinBox: Segment count selector
  • QPushButton: Clear Completed, Open Download Folder, Close

8. Troubleshooting

Download never starts:

  • Check internet connection
  • Verify the URL is accessible
  • Check if the server requires authentication (not supported yet)
  • Look at terminal output for network errors

Segmented download fails to merge:

  • Check disk space in download directory
  • Verify write permissions to the download folder
  • Segment files are preserved in .dlcache; you can retry the download
  • If segments are corrupted, delete the .dlcache folder and retry

Resume not working after browser restart:

  • Check that the .onudl index file exists in ~/Downloads/.dlcache
  • Verify segment files still exist in .dlcache
  • The server must still support range requests for resume to work
  • Some servers expire range request capability after a period of time

Smart path creating wrong folders:

  • Check file extension detection (case-insensitive)
  • Files with no extension go to "Others" folder
  • Unknown extensions also go to "Others"
  • The folder structure is relative to your configured download path

Progress bar stuck at 0%:

  • Server may not be sending Content-Length header (unknown size)
  • Progress bar becomes indeterminate in this case (animated)
  • Speed and downloaded size still update correctly

Downloads disappear from manager after restart:

  • Check that the .onudl file was written successfully
  • Verify the download directory hasn't been deleted or moved
  • Completed downloads that no longer exist on disk are not restored

File not found error on completed download:

  • The file may have been moved or deleted outside Onu
  • The UI shows strikethrough text and "File not found" status
  • Use "Show in Folder" to locate the file or remove the entry

High memory usage during segmented downloads:

  • Each segment uses a separate QNetworkAccessManager instance
  • With 32 segments, this can use significant memory
  • Reduce segment count if you experience memory issues
  • Files are written to disk as data arrives, not kept entirely in memory

The Onu downloader is designed to be robust, efficient, and user-friendly. With segmented downloads for speed, and the indexed storage ensures you never lose track of what you have downloaded.
